Validation Rules Reference

Rule Usage Description
required data-validate="required" Field must not be empty
email data-validate="email" Must be valid email format
phone data-validate="phone" Rwandan phone format (0XXXXXXXXX)
positiveNumber data-validate="positiveNumber" Must be greater than 0
min:value data-validate="min:10" Minimum value or length
max:value data-validate="max:100" Maximum value or length
minLength:n data-validate="minLength:3" Minimum character length
maxLength:n data-validate="maxLength:50" Maximum character length
tin data-validate="tin" 9-digit Tax ID (optional)
password data-validate="password" Minimum 6 characters
match:fieldId data-validate="match:password" Must match another field
date data-validate="date" Valid date format
pastDate data-validate="pastDate" Date must be in past
Multiple Rules: Combine rules using pipe (|) separator. Example: data-validate="required|phone"
